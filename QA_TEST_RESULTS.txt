BUG 15.1: System Events Had NULL Master Log Lines
  DESCRIPTION: When a day was reopened, the system-generated log event 
    ("Day reopened by John Smith") was created with category "directive" 
    but no corresponding log_renders records were created.
    This caused the event to display with:
      - masterLogLine: null
      - aiStatus: null
      - internalLine: null

  ROOT CAUSE: The reopen endpoint in server/routes.ts created the log 
    event via storage.createLogEvent() but did not create any 
    log_renders records for the system event.

  FIX APPLIED:
    1. Changed system event category from "directive" to "ops"
    2. Added explicit creation of log_renders records:
       - master_log_line: "At HH:MM, Day reopened by <user>."
       - internal_canvas_line: "**HH:MM | SYSTEM:** Day reopened by <user>"
    3. Backfilled 7 existing orphaned events in the database

  VERIFICATION: 
    - 0 orphaned log events remaining (confirmed via SQL query)
    - All system events now show proper masterLogLine and aiStatus: "ok"
